01 — ROLES

Three agents, one job each

Each tool does the one thing it's actually best at. No tool tries to do another's job.

Researcher

Perplexity

Runs live web searches and returns sourced, current findings. Not limited by a training cutoff — it's reading today's web, today.

Analyst

Claude

Takes raw search output and turns it into something you'd actually want to read — structured, prioritized, and stripped of noise.

Operator

Make.com

Triggers the whole run on a schedule and passes data between the other two. This is what makes it autopilot instead of a manual task.

02 — BUILD

The scenario, module by module

Build one Make.com scenario with these four modules in order. Each one's output maps directly into the next one's input.

1

Trigger — Schedule

Built into Make. Set it to run daily, weekly, or whatever cadence fits your topic. This is the only step you interact with after setup — and only if you want to change the timing.

2

HTTP → Perplexity API

Perplexity has no native Make app, so use the HTTP module to call it directly.

  • Method: POST → https://api.perplexity.ai/chat/completions
  • Header: Authorization: Bearer YOUR_PERPLEXITY_API_KEY
{
  "model": "sonar-pro",
  "messages": [
    {
      "role": "user",
      "content": "Find the most important news and developments from the last 24 hours on [YOUR TOPIC]. Include sources."
    }
  ]
}
Output text feeds directly into Module 3
3

Anthropic → Create a Message

Make has a native Claude app — connect your API key and select claude-sonnet-4-6.

You are a research analyst. You will receive raw web search results.
Turn them into a clean summary with:
1. A 2-sentence executive summary at the top
2. 3-6 bullet points of the most important findings
3. Source links preserved
4. Flag anything time-sensitive or requiring action
Keep it under 400 words. No fluff.

User message: map in the raw text output from Module 2.

Claude's reply is your finished summary — map it into Module 4
4

Delivery

Pick one, or run two in parallel from the same scenario:

  • Email — Gmail/Outlook module, Claude's output as the body
  • Slack — post to a channel or DM yourself
  • Notion — logs each run as a searchable archive page

Most people run email for the daily nudge and Notion as the permanent archive.

04 — LATER

Ways to extend this

Run multiple Perplexity queries — one per topic — feeding into a single combined Claude summary.
Add a router module to send different topics to different Slack channels.
Add a second Claude call that scores each finding 1–10 for relevance and filters out the low scores.
Archive every run in Notion or Airtable to build a searchable history over time.
